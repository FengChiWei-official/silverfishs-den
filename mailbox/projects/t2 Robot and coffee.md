## Text
既然每个机器人的药水使用上限 $L_i$ 是不同的（即不再是统一的 $n$），这其实让问题变得更具**针对性**，但解题的核心框架依然是 **“0/1背包 + 贪心排序”** 的组合。

以下是针对“个性化上限”优化后的要点总结和题目规范表述。

### 1. 核心解题要点

1.  **目标：** 最大化总节省时间（$\sum \Delta T_i$）。
2.  **分类处理：**
    *   **不可拆分机器人（Type B）**：
        *   每个机器人是一个“0/1 背包项目”。
        *   它的体积是 $n_i$，价值是 $m_i$。
        *   **约束检查**：如果 $n_i > L_i$（消耗超过了它自己的上限），或者 $m_i > T_i$（节省时间超过了总时间，除非允许任务时间为负数），则该机器人无法被加速，直接排除。
    *   **可拆分机器人（Type A）**：
        *   每个机器人是一个“分数背包项目”。
        *   计算单位效率 $r_i = m_i / n_i$。
        *   **有效上限判定**：它能使用的最大药水量 $P_{max, i} = \min(L_i, T_i / r_i)$。
3.  **算法流程：**
    *   **第一步 (DP)**：对所有类型 B 机器人跑一个 $0/1$ 背包，得到数组 $dp[j]$，表示用 $j$ 瓶药水在这一堆人里能省下的最大时间。
    *   **第二步 (贪心)**：对所有类型 A 机器人按 $r_i$ 从高到低排序。
    *   **第三步 (整合)**：枚举分配。总收益 = $\max (dp[j] + \text{贪心分配剩余的 } N-j \text{ 瓶给类型 A})$。

---

### 2. 标准题目规范表述

#### **题目名称**：异构约束下的多机器人任务并行加速分配优化

#### **问题描述**：
给定 $K$ 个机器人，每个机器人正在执行一个独立的平行任务，初始耗时为 $T_i$。现有总量为 $N$ 的加速药水。每个机器人 $i$ 对药水的使用有其特定的限制和增益逻辑。请设计方案分配药水，使得所有任务的**总完成时间之和**（$\sum \text{Final } T_i$）最小。

#### **机器人属性与约束**：
1.  **个体上限约束**：每个机器人 $i$ 最多只能使用 $L_i$ 瓶药水。
2.  **加速逻辑分类**：
    *   **连续加速型 (Type A)**：药水使用量 $x_i$ 可取任意实数，满足 $0 \le x_i \le L_i$。每使用 $n_i$ 瓶药水可减少 $m_i$ 时间。即节省时间 $\Delta T_i = x_i \cdot \frac{m_i}{n_i}$。
    *   **阶跃加速型 (Type B)**：药水使用具有原子性。只能选择分配 $x_i = n_i$ 瓶或 $x_i = 0$ 瓶。前提是必须满足 $n_i \le L_i$。若分配 $n_i$ 瓶，节省时间 $\Delta T_i = m_i$；否则为 0。
3.  **时间非负约束**：对于任何机器人，节省的时间 $\Delta T_i$ 不应超过其初始时间 $T_i$。

#### **输入定义**：
*   $N$: 可用的药水总数。
*   机器人集合 $\{R_1, R_2, \dots, R_K\}$，其中每个机器人包含：
    *   $Type_i$: 机器人类型（A 或 B）。
    *   $T_i$: 任务初始时长。
    *   $L_i$: 该机器人药水承载上限。
    *   $n_i$: 加速基准药水量。
    *   $m_i$: 加速基准时间收益。

#### **优化目标**：
$$\text{Maximize } \sum_{i=1}^{K} \Delta T_i$$
在满足以下约束的情况下：
1. $\sum_{i=1}^{K} x_i \le N$
2. 对于 Type A: $0 \le x_i \le \min(L_i, \frac{T_i \cdot n_i}{m_i})$
3. 对于 Type B: $x_i \in \{0, n_i\}$ 且 $x_i \le L_i$ 且 $m_i \le T_i$

#### **算法参考**：
本问题属于**混合背包问题（Mixed Knapsack Problem）**的变体。可以通过**动态规划（Dynamic Programming）**处理阶跃型变量，并结合**贪心策略（Greedy Strategy）**处理连续型变量，最终通过线性扫描合并最优解。
